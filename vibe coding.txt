Bangun sebuah aplikasi web platform pesan anonim seperti Secreto atau NGL menggunakan Next.js 14 App Router dengan TypeScript sebagai frontend dan Supabase sebagai backend service untuk database, API, dan realtime update, dengan konsep anonymous message board dimana siapa pun bisa langsung mengirim pesan tanpa perlu login atau daftar lalu pesan tersebut otomatis muncul di halaman homepage sebagai feed publik sehingga bisa dibaca oleh semua pengunjung dan siapa pun juga bisa membalas pesan tersebut termasuk pengunjung lain maupun saya sebagai owner, sehingga sistemnya terasa seperti anonymous confession board yang memiliki thread diskusi; fokuskan pembuatan UI yang modern, interaktif, mobile first, responsif, dan user friendly dengan visual clean namun playful menggunakan TailwindCSS dan Shadcn UI sebagai komponen utama, gunakan font Poppins sebagai font utama, Inter sebagai font pendukung, dan JetBrains Mono untuk elemen teknis atau UI tertentu, gunakan Lucide React sebagai library icon utama ditambah Heroicons atau React Icons jika dibutuhkan agar tampilan icon lebih kaya, serta gunakan Framer Motion untuk memberikan animasi halus seperti hover card animation, smooth card appearance, message reveal animation, page transition, dan micro interaction agar UI terasa hidup dan tidak kaku; halaman utama homepage harus memiliki hero section dengan judul “Send Anonymous Message” dan subtext “Say anything. No name. No identity.” serta tombol Send Message yang mengarah ke halaman pengiriman pesan, kemudian di bawahnya terdapat message feed yang menampilkan semua pesan anonim dalam bentuk card seperti social media feed dimana setiap card menampilkan avatar anonim random menggunakan DiceBear Avatar API, label Anonymous, isi pesan, jumlah balasan, timestamp, serta tombol Reply untuk membuka thread diskusi; setiap pesan dapat memiliki reply thread sehingga balasan bisa berantai menggunakan sistem parent reply sehingga diskusi bisa berkembang, dan semua orang dapat membalas tanpa perlu login, namun khusus untuk saya sebagai owner ketika membalas pesan maka balasan tersebut harus memiliki badge atau logo khusus seperti label Owner atau icon crown sehingga pengguna lain tahu bahwa pesan tersebut dijawab oleh owner; buat halaman /send yang berisi form sederhana untuk mengirim pesan anonim dengan input message maksimal 500 karakter dan tombol Send anonymously tanpa proses registrasi sama sekali, lalu buat halaman /dashboard khusus owner untuk mengelola pesan dengan fitur seperti delete message, pin message, highlight message, filter message, dan search message, dashboard ini bisa menggunakan sistem login sederhana menggunakan Supabase Auth atau environment key untuk akses owner; desain feed menggunakan grid responsif yaitu 1 kolom di mobile, 2 kolom di tablet, dan 3 kolom di desktop dengan background gradient modern bernuansa purple, pink, dan blue serta card menggunakan efek glassmorphism dengan blur ringan dan soft shadow agar terlihat modern, serta tambahkan fitur dark mode dan light mode yang konsisten sehingga tidak ada teks yang hilang atau tidak terbaca ketika mode berubah; struktur project Next.js minimal terdiri dari folder app yang berisi page.tsx sebagai homepage, send/page.tsx untuk halaman kirim pesan, dan dashboard/page.tsx untuk owner panel, lalu folder components berisi MessageCard.tsx, ReplyCard.tsx, MessageForm.tsx, dan Navbar.tsx, folder lib berisi supabase.ts dan api.ts untuk koneksi Supabase, folder hooks berisi useMessages.ts untuk fetch dan realtime subscription, serta folder types untuk mendefinisikan tipe data message dan reply; gunakan Supabase realtime agar ketika pesan baru atau balasan baru masuk maka feed langsung terupdate tanpa refresh halaman, tambahkan juga fitur tambahan seperti emoji reaction pada pesan, spam protection misalnya pembatasan pengiriman pesan satu kali setiap 30 detik untuk mencegah spam, random avatar generator untuk setiap pesan anonim, serta UI interaction seperti hover highlight, smooth reply expand, dan animated card entry agar pengalaman pengguna terasa seperti aplikasi sosial modern.
